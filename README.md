## Что проверить и понять

### 1. Закон Ома (строка `const I = U / R`)
Это ядро всей программы — одна строчка. Убедись, что понимаешь: ток растёт при увеличении напряжения и падает при увеличении сопротивления. Попробуй изменить формулу на `U * R` — и посмотри, что сломается.

### 2. Считывание ползунков
```js
const U = parseFloat(sliderU.value);
```
Значение ползунка всегда приходит как **строка** (`"12"`), поэтому нужен `parseFloat`. Проверь: что будет, если убрать `parseFloat`?

### 3. Обновление страницы
```js
sliderU.addEventListener('input', update);
```
Событие `input` срабатывает при **каждом движении** ползунка. Замени на `'change'` — и ток будет пересчитываться только после того, как отпустишь мышь. Почувствуй разницу.

### 4. Координатные функции в графике
```js
function xPx(u) { return padL + (u / 24) * plotW; }
function yPx(i) { return padT + plotH - (i / Imax) * plotH; }
```
Это перевод физических единиц (В, А) в пиксели. Обрати внимание: в `yPx` есть `plotH - (...)` — потому что на экране ось Y направлена **вниз**, а в физике — вверх.

### 5. Линейный график
```js
ctx.moveTo(xPx(0), yPx(0));
ctx.lineTo(xPx(24), yPx(24 / R));
```
`I = U/R` — линейная функция, поэтому достаточно двух точек. Измени R — и наклон прямой изменится. Чем больше R, тем **положе** прямая.

### 6. Жёлтая точка
Она всегда показывает твоё текущее положение на графике. Пунктирные линии — проекции на оси, как на уроке физики. Найди в коде `ctx.arc(...)` — это команда рисования круга.
